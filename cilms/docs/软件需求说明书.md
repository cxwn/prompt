# CILMS 软件需求说明书

> Container/Init Lifecycle Management System — 容器与进程生命周期管理系统

## 1. 项目概述

CILMS 是一款运行于 Linux 操作系统下的进程管理工具，适用于容器环境（作为 1 号进程）及非 root 用户的物理机/虚拟机环境。系统负责受控进程的启动、停止、重启、监控、日志管理等全生命周期管理，并提供 CLI 命令行和 HTTP API 两种交互方式。

## 2. 功能需求

### 2.1 进程管理

#### 2.1.1 运行模式

- **容器环境**：作为容器内的 1 号进程（PID=1）启动，强制以前台模式运行。若指定了守护进程参数则忽略并输出警告日志。自身完成所有进程启动后阻塞容器不退出。
- **非容器环境**：支持守护进程模式和前台运行模式。
- 两种场景下，均作为受控进程的父进程运行。

#### 2.1.2 容器环境检测

通过以下方式判断是否运行在容器中：
- 检查当前进程 PID 是否为 1；
- 检测 `/.dockerenv` 文件是否存在；
- 检测 `/proc/1/cgroup` 是否包含容器特征。

#### 2.1.3 进程启动顺序

- 根据配置中的 `priority` 字段按顺序启动受控进程（数字越小优先级越高）；
- 若优先级未配置或多个进程优先级值相同，则按配置文件中进程配置序号启动；
- 受控进程可以同名（例如需要启动多个 nginx 进程），系统内部通过配置序号生成唯一标识（如 `nginx-1`、`nginx-2`）；
- `dependsOn` 引用同名进程时表示依赖该名称的所有实例；
- 启动时对 `dependsOn` 进行 DAG 拓扑排序校验，检测到循环依赖时拒绝启动并输出错误日志，明确指出循环依赖链。

### 2.2 进程监控

#### 2.2.1 僵尸进程与孤儿进程管理

**PID 1 模式（容器环境）：**
- CILMS 作为 init 进程，负责回收所有子进程（包括受控进程及被托管的孤儿进程）的僵尸状态；
- 检测到孤儿进程（即父进程已终止而被重新托管至 CILMS 的进程）时，通过 `waitpid()` 回收其资源并记录日志。

**非 PID 1 模式（普通环境）：**
- CILMS 仅能回收自身直接启动的受控子进程的僵尸状态；
- 无法接收系统中其他孤儿进程的托管；
- `cilms orphans` 命令在此模式下仅报告受控子进程范围内的异常状态。

**通用规则：**
- 两种模式下，检测到受控进程的僵尸状态后，立即通过 `waitpid()` 回收其退出状态，再根据该进程的重启策略决定是否重新启动。

#### 2.2.2 运行状态记录

- 监控并记录受控子进程的运行状态：运行中、停止、重启次数、退出码等；
- 监控并记录受控子进程的启动时间、停止时间、重启时间等。

### 2.3 重启策略

#### 2.3.1 重启模式

| 策略 | 说明 |
|------|------|
| `always` | 默认值。无论进程因何原因终止（正常退出或异常退出），均自动重启 |
| `onFailure` | 仅当进程以非零退出码终止时（异常退出）才重启，正常退出（退出码为 0）不触发重启 |
| `never` | 无论进程终止状态如何，不自动重启，除非手动重启 |

#### 2.3.2 指数退避策略

启动失败后按指数退避策略重启：
- 第 1 次：延迟 5 秒
- 第 2 次：延迟 10 秒
- 第 3 次：延迟 20 秒
- 第 4 次：延迟 40 秒
- 以此类推，最大延迟不超过 300 秒

#### 2.3.3 最大重启次数

- 超过配置文件中的 `maxRetries` 最大次数后，根据 `critical` 字段判断：
  - `critical: true`：容器环境中退出 1 号进程，其他环境中退出 CILMS 进程；
  - `critical: false`（默认值）：将该进程标记为 `failed` 状态并记录错误日志，不影响其他进程运行。

#### 2.3.4 退避重置

如果在某次重启后能稳定运行 10 分钟以上，后续若再需要重启，将重新从 5 秒开始计算延迟时间。

#### 2.3.5 依赖感知重启

如果进程依赖其他进程，则需确保被依赖的进程状态正常后再重启。进程"状态正常"的判定规则：
- 若配置了 `healthCheck`，则以健康检查通过为准；
- 若未配置，则以进程处于 `running` 状态为准。

### 2.4 进程停止

- 优雅关闭：发送 `SIGTERM` 或 `SIGINT` 信号；
- 超时则强制终止：发送 `SIGKILL` 信号；
- 按与启动优先级相反的顺序停止受控进程；
- CILMS 自身收到 `SIGTERM`/`SIGINT` 信号时，将信号转发给所有受控子进程，并按优先级反序依次优雅关闭。

### 2.5 配置热加载

CILMS 收到 `SIGHUP` 信号时，重新读取配置文件并应用非破坏性变更，包括：
- 日志级别调整；
- 新增受控进程；
- 移除已停止的受控进程配置。

已运行进程的参数变更不会自动生效，需手动重启对应进程。

### 2.6 日志管理

#### 2.6.1 受控进程日志管理

针对原应用系统开发时无日志轮转机制、可能造成磁盘空间耗尽的问题，若配置文件内指定了受控进程的日志处理配置（`logRotate`），则支持：
- 日志文件自动轮转
- 日志压缩
- 日志归档

#### 2.6.2 系统自身日志

- 支持日志文件输出至文件和控制台（可同时输出）；
- 支持日志文件的轮转、压缩、归档；
- 日志级别：`DEBUG`、`INFO`、`WARN`、`ERROR`、`FATAL` 共 5 个级别；
- 日志级别固定字符数宽度为 5，左对齐；
- 日志框架二次封装后提供终端颜色配置。

## 3. 命令行接口设计

CILMS CLI 采用客户端-服务端架构：`start` 命令启动 daemon 进程并监听 HTTP 端口；其他子命令（`stop`、`restart`、`list`、`zombies`、`orphans`、`logs`）通过 HTTP 请求连接 `127.0.0.1:{port}` 与运行中的 daemon 通信。若 daemon 未启动或端口不可达，则输出错误提示并退出。默认端口为 `10168`，可通过 `-p` 参数或配置文件指定。

### 3.1 启动命令

```bash
cilms start -d -p 10168 -c /etc/cilms/cilms.yaml
```

| 参数 | 长参数 | 说明 |
|------|--------|------|
| `-d` | `--daemon` | 守护进程模式，不指定则前台运行 |
| `-p` | `--port` | 监听端口，默认 10168 |
| `-c` | `--conf` | 配置文件路径，默认 `/etc/cilms/cilms.yaml` |

启动成功返回启动成功信息，否则返回启动失败信息。

当 CILMS daemon 已运行时，可启动单个已停止的受控进程：

```bash
cilms start -n nginx-1
```

| 参数 | 长参数 | 说明 |
|------|--------|------|
| `-n` | `--name` | 启动指定内部唯一标识的已停止受控进程 |

### 3.2 停止命令

停止 CILMS 进程及所有受控进程：

```bash
cilms stop
```

`stop` 命令会等待所有子进程退出后退出。若成功则输出停止成功信息，否则输出停止失败信息。

停止指定受控进程：

```bash
cilms stop -n nginx-1
```

| 参数 | 长参数 | 说明 |
|------|--------|------|
| `-n` | `--name` | 停止指定内部唯一标识的受控进程 |

### 3.3 重启命令

重启 CILMS 进程及所有进程：

```bash
cilms restart
```

按启动优先级重启所有进程。若成功则输出重启成功信息，否则输出重启失败信息。

重启指定受控进程：

```bash
cilms restart -n nginx-1
```

| 参数 | 长参数 | 说明 |
|------|--------|------|
| `-n` | `--name` | 重启指定内部唯一标识的受控进程 |

### 3.4 状态查询命令

```bash
cilms list -a
cilms list -n nginx-1
```

| 参数 | 长参数 | 说明 |
|------|--------|------|
| `-a` | `--all` | 列出所有进程的状态（CILMS ID、进程名称、状态、重启次数等） |
| `-n` | `--name` | 查询指定内部唯一标识的受控进程状态 |

选项 `-a` 与 `-n` 不能同时使用。

### 3.5 僵尸进程检测命令

```bash
cilms zombies          # 列出僵尸进程
cilms zombies -c       # 清理僵尸进程
cilms zombies -c -r    # 清理僵尸进程并重启对应的受控进程
```

| 参数 | 长参数 | 说明 |
|------|--------|------|
| `-c` | `--clean` | 执行僵尸进程清理操作，不指定则仅列出 |
| `-r` | `--restart` | 重启被清理的对应受控进程，默认不重启 |

### 3.6 孤儿进程检测命令

```bash
cilms orphans          # 列出孤儿进程
cilms orphans -c       # 清理孤儿进程
cilms orphans -c -r    # 清理孤儿进程并重启对应的受控进程
```

| 参数 | 长参数 | 说明 |
|------|--------|------|
| `-c` | `--clean` | 执行孤儿进程清理操作，不指定则仅列出 |
| `-r` | `--restart` | 重启被清理的对应受控进程，默认不重启 |

### 3.7 日志查询命令

查看 CILMS 自身日志：

```bash
cilms logs -f -n 100
```

查看指定受控进程的日志：

```bash
cilms logs --id nginx-1 -f -n 100
```

| 参数 | 长参数 | 说明 |
|------|--------|------|
| `-f` | `--follow` | 实时跟踪日志输出，不指定则仅显示最后 N 行 |
| `-n` | `--number` | 指定日志条数，默认 100 |
| | `--id` | 指定受控进程的内部唯一标识，不指定则显示 CILMS 自身日志 |

## 4. 系统日志格式

- 支持 `json` 和 `text` 两种格式，默认 `text`，可配置；
- 日志同时输出到文件和控制台，两种格式均可独立配置；
- 日志级别：`DEBUG`、`INFO`、`WARN`、`ERROR`、`FATAL`，默认 `INFO`。配置时大小写均可；
- Text 格式示例（时间为当前系统时间）：

```text
2006-01-02T15:04:05.000+08:00 INFO  12345 cilms/main.go:100 msg
```

其中时间格式为 Go 的 time layout 参考值（ISO 8601 / RFC 3339，含时区偏移），`12345` 为 CILMS 自身进程的 PID。

## 5. HTTP 接口设计

### 5.1 统一响应格式

成功响应：

```json
{
  "status": 200,
  "message": "Success",
  "data": { /* 接口返回的具体数据 */ }
}
```

错误响应：

```json
{
  "status": 400,
  "message": "Error description",
  "error": "Detailed error reason (optional)"
}
```

### 5.2 接口列表

#### 5.2.1 获取所有进程列表

- **路径**: `/cilms/processes`
- **方法**: `GET`
- **描述**: 返回系统中所有受控进程的列表

```http
GET /cilms/processes HTTP/1.1
```

#### 5.2.2 获取指定受控进程信息

- **路径**: `/cilms/processes/{id}`
- **方法**: `GET`
- **描述**: 返回指定内部唯一标识对应的受控进程详细信息
- **URL 参数**: `id`（字符串）— 受控进程的内部唯一标识（如 `nginx-1`）

```http
GET /cilms/processes/nginx-1 HTTP/1.1
```

#### 5.2.3 重启指定受控进程

- **路径**: `/cilms/processes/{id}/restart`
- **方法**: `POST`
- **描述**: 重启指定内部唯一标识对应的受控进程
- **URL 参数**: `id`（字符串）

```http
POST /cilms/processes/nginx-1/restart HTTP/1.1
```

#### 5.2.4 停止指定受控进程

- **路径**: `/cilms/processes/{id}/stop`
- **方法**: `POST`
- **描述**: 停止指定内部唯一标识对应的受控进程
- **URL 参数**: `id`（字符串）

```http
POST /cilms/processes/nginx-1/stop HTTP/1.1
```

#### 5.2.5 启动指定受控进程

- **路径**: `/cilms/processes/{id}/start`
- **方法**: `POST`
- **描述**: 启动指定内部唯一标识对应的已停止受控进程
- **URL 参数**: `id`（字符串）

```http
POST /cilms/processes/nginx-1/start HTTP/1.1
```

#### 5.2.6 停止所有进程

- **路径**: `/cilms/stop`
- **方法**: `POST`
- **描述**: 停止系统中所有运行的进程

```http
POST /cilms/stop HTTP/1.1
```

#### 5.2.7 重启所有进程

- **路径**: `/cilms/restart`
- **方法**: `POST`
- **描述**: 重启系统中的所有进程

```http
POST /cilms/restart HTTP/1.1
```

#### 5.2.8 获取僵尸进程列表

- **路径**: `/cilms/zombies`
- **方法**: `GET`
- **描述**: 返回系统中所有僵尸进程的列表

```http
GET /cilms/zombies HTTP/1.1
```

#### 5.2.9 清理僵尸进程

- **路径**: `/cilms/zombies/clean`
- **方法**: `POST`
- **描述**: 清理系统中的所有僵尸进程
- **查询参数**: `restart`（布尔值，可选）— 是否在清理后重启对应的受控进程，默认 `false`

```http
POST /cilms/zombies/clean?restart=true HTTP/1.1
```

#### 5.2.10 获取孤儿进程列表

- **路径**: `/cilms/orphans`
- **方法**: `GET`
- **描述**: 返回系统中所有孤儿进程的列表

```http
GET /cilms/orphans HTTP/1.1
```

#### 5.2.11 清理孤儿进程

- **路径**: `/cilms/orphans/clean`
- **方法**: `POST`
- **描述**: 清理系统中的所有孤儿进程
- **查询参数**: `restart`（布尔值，可选）— 是否在清理后重启对应的受控进程，默认 `false`

```http
POST /cilms/orphans/clean?restart=true HTTP/1.1
```

所有接口均返回 JSON 格式的响应数据。

## 6. 配置文件规范

配置文件格式为 YAML，默认路径 `/etc/cilms/cilms.yaml`。

### 6.1 全局配置

```yaml
global:
  port: 10168                    # HTTP 监听端口
  bind: "127.0.0.1"             # 监听地址，默认仅本地
  token: ""                      # API 访问令牌，为空表示无认证；设置后请求须携带 Authorization: Bearer <token>
  pidFile: "/var/run/cilms.pid"  # PID 文件路径，守护进程模式写入，退出时清理，启动时检查防止重复实例
  logger:
    file: "logs/cilms.log"       # 日志文件路径
    level: "debug"               # 日志级别：debug, info, warn, error, fatal
    rotate: true                 # 是否启用日志轮转
    maxSize: 100                 # 单个日志文件最大大小（MB）
    maxBackups: 5                # 保留的旧日志文件数量
    maxAge: 30                   # 日志文件保留天数
    compress: true               # 是否压缩旧日志
    console: true                # 是否同时输出到控制台
    format: "text"               # 日志格式：text 或 json
```

### 6.2 进程配置

```yaml
processes:
  - name: "demo"                          # 进程名称（可重复，系统自动生成唯一标识）
    environment:                           # 环境变量（key: value 映射）
      PORT: "8080"
      DB_HOST: "localhost"
    dependsOn: ["rotate", "zombie"]        # 依赖列表（可选）
    directory: "/cilms/demo"               # 工作目录
    command: "demo"                        # 启动命令
    args: ["--port", "${PORT}"]            # 命令参数，${VAR} 从进程环境 > 系统环境解析，支持 ${VAR:-default} 默认值
    priority: 3                            # 启动优先级，数字越小优先级越高
    restartPolicy: "always"                # 重启策略：always, onFailure, never
    maxRetries: 5                          # 最大重试次数
    critical: true                         # 是否关键进程（超过 maxRetries 时决定是否退出 CILMS）
    stopTimeout: 10                        # 优雅关闭超时秒数（超时后发送 SIGKILL）
    startTimeout: 30                       # 启动超时秒数（超时标记为 failed）
    healthCheck:                           # 健康检查配置（可选）
      type: "http"                         # 检查类型：http, tcp, exec
      target: "http://127.0.0.1:8080/health"  # http 为 URL，tcp 为地址，exec 为命令
      interval: 10                         # 检查间隔（秒）
      timeout: 5                           # 单次检查超时（秒）
      retries: 3                           # 失败重试次数
      initialDelay: 5                      # 首次检查延迟（秒）
    logRotate:                             # 受控进程日志轮转配置
      rotate: true                         # 是否启用
      file: "/var/log/demo/example.log"    # 受控进程日志文件路径
      maxSize: 128                         # 单个日志文件最大大小（MB）
      maxDays: 3                           # 日志保留天数
      maxBackups: 5                        # 保留的旧日志文件数量
      compress: true                       # 是否压缩
      checkInterval: 60                    # 轮转检查间隔（秒）
```

## 7. 技术选型

| 类别 | 选型 |
|------|------|
| 开发语言 | Go 1.24.2 |
| 命令行库 | `urfave/cli/v3` |

其他有更好的技术方案可灵活选用。

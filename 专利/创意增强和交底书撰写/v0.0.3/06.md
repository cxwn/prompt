#  step 8：解析模板并编写交底书（核心输出）
## 8.1 模板解析与写作约束（强制）
- **解析模板中已给出的全部写作要求**
- 严格按模板结构与约束编写：
  - 不擅自增删章节
  - 不重命名标题
  - 不改变章节顺序
- 所有技术内容必须：
  - 可工程实现
  - 可被审查员理解与复核
  - 可直接拆解为权利要求
---

## 8.2 交底书技术写作规范（新增，强制执行）
交底书中涉及**技术定义、数学表达、模型描述、规则推导、控制逻辑**时，**必须采用“工程化 + 审查友好”的规范写作方式**，具体要求如下：
### （1）技术概念定义要求
- 所有关键技术对象 **首次出现必须明确定义**
- 定义应至少包含：
  - 技术含义
  - 所属系统或模块
  - 功能作用与使用位置
- 禁止以下不合规表述：
  - 仅描述效果、不描述机制
  - 使用“智能地 / 动态地 / 自动地”等词汇而无技术限定
### （2）公式 / 规则 / 推导类内容要求（含 LaTeX 强制规范）
- 凡涉及以下内容之一，**必须使用数学或形式化表达**：
  - 计算公式
  - 决策规则
  - 调度 / 评估 / 优化函数
  - 模型输入输出关系
- **所有公式必须使用 LaTeX 语法输出**，不得使用自然语言或伪代码替代
- 每一个公式必须同时给出：
  - 完整 LaTeX 表达式
  - 所有符号的逐一说明（含单位或取值范围）
  - 适用条件、边界或约束前提
- 数学或规则表达必须明确支撑：
  - 技术手段如何实施
  - 技术效果为何能够达成
### （3）表达风格与 LaTeX 示例（必须对齐）
以下示例为**交底书允许且推荐的技术表达粒度、逻辑结构与 LaTeX 书写规范**：
```
物体沿直线作匀加速运动时，其位移定义为运动终止时刻位置坐标与初始位置坐标之差值，为具有方向性的代数量；该位移$s$由初始速度$v_0$、恒定加速度$a$及运动时间$t$唯一确定，满足表达式：
$$s = v_0 t + \frac{1}{2} a t^2$$
其中，$v_0$ 为初始速度（单位：m/s），$a$ 为恒定加速度（单位：m/s²）且方向与运动轨迹共线，$t$ 为运动持续时间（单位：s，且 $t \ge 0$），$s$ 为位移（单位：m）当 $s > 0$ 时，位移方向与预设正方向一致；当 $s < 0$ 时，位移方向相反；上述公式适用于加速度恒定且运动轨迹为直线的运动场景。
```
### （4）在专利交底书中的等价应用方式
在专利交底书中，上述写作方式必须等价映射为：
* **系统行为 → 参数定义 → LaTeX 规则 / 函数 → 技术效果**
* 示例映射关系：
  * “位移 $s$” → 系统输出结果
  * “$v_0$、$a$” → 输入参数 / 控制变量
  * “公式” → 决策逻辑 / 调度函数 / 评估模型
  * “适用条件” → 系统运行环境、工程约束或边界条件
### （5）流程图 / 架构图表达要求（Mermaid 强制规范）
- 凡涉及以下内容之一，**必须提供结构化图示**：
  - 系统总体架构
  - 模块间交互关系
  - 数据流 / 控制流
  - 算法或决策流程
  - 方法步骤执行顺序
- **所有流程图、架构图必须使用 Mermaid 语法输出**
- 禁止：
  - 仅用自然语言描述复杂流程
  - 使用图片占位、口头化说明替代图示
- 每一个 Mermaid 图必须：
  - 与正文描述一一对应
  - 节点名称与技术对象命名保持一致
  - 明确输入、处理、输出或控制关系
### （6）Mermaid 表达示例（必须对齐）
**示例一：方法流程图**
```mermaid
flowchart TD
    A[接收系统运行指标数据] --> B[特征提取模块]
    B --> C[异常评估函数计算]
    C -->|超过阈值| D[触发调度控制策略]
    C -->|未超过阈值| E[维持当前运行状态]
````
**示例二：系统架构图**
```mermaid
graph LR
    User[运维人员] --> API[控制接口模块]
    API --> Engine[智能决策引擎]
    Engine --> Model[评估模型模块]
    Engine --> Executor[执行控制模块]
    Executor --> System[被控系统]
```
### （7）LaTeX + Mermaid 的专利交底书等价映射原则
在专利交底书中必须形成如下对应关系：
* **自然语言描述**：说明技术背景与整体目的
* **Mermaid 图**：限定系统结构、流程边界与模块关系
* **LaTeX 公式 / 规则**：限定核心技术手段与决策机制
三者必须：
* 逻辑一致
* 相互引用
* 不出现语义冲突或范围扩张
